一、Concordia 是什么

一句话：Concordia 是一个团队级 AI 工作系统——AI 主动发现每个人的待办，提前准备好草稿和方案，让团队成员只需确认和决策。

和 ClawPilot 的区别：ClawPilot 帮一个人做事，Concordia 帮一个团队协作。ClawPilot 是个人助手，Concordia 是团队操作系统。

核心价值：

  1. AI 主动发现 — 不需要你告诉 AI 要做什么，AI 从邮件/Teams/日历中自动发现
  2. AI 提前准备 — 不只是提醒你"有事要做"，而是帮你把回复草拟好、方案整理好、简报准备好
  3. 团队可见 — 不只是你自己看到，团队里谁被卡住了、哪些事有关联，AI 自动发现并通知相关人

二、产品架构

整体结构

用户看到的
├── 首页：我的待办（AI 已预处理）+ 团队动态 + 日程 + 洞察
├── 工作台：项目空间（知识 / 文件 / 讨论摘要 / 团队任务）
├── 会议：日历 + AI 会前简报
└── 右侧 AI 面板：点击任何内容展示上下文 + 操作建议

底层引擎
├── 感知层：Pipeline 实时监听 Teams / 邮件 / 日历
├── 理解层：信号过滤 + 角色适配 + 跨人关联
├── 准备层：AI 预处理（草拟回复 / 准备简报 / 起草方案）
├── 协作层：Team Skills（团队协作 SOP）
└── 学习层：用户风格学习 + 团队经验沉淀

每一层做什么

感知层：从哪来数据

信源怎么监听产出
TeamsPipeline YAML，监听 DM + @mention发现"有人让我做事"的信号
邮件Pipeline YAML，排除 noreply/通知类发现需要回复/行动的邮件
日历Pipeline YAML，会前 5-30 分钟触发生成会前准备提醒
ADO定时扫描分配给我的工作项

已实现：3 个 Pipeline YAML 已部署，今天已触发过 7+ 次。

理解层：哪些信号需要处理

四问过滤（每条信号必须全过）：

  1. 有人明确要我做具体动作吗？→ FYI/闲聊丢弃
  2. 我不做会有后果吗？→ 可选参与丢弃
  3. 现在还需要我行动吗？→ 已过期丢弃
  4. 上级在讨论和我相关的重要方向吗？→ 即使没 @我也纳入

角色适配（自动检测用户级别）：

角色判定条件看到什么
IC / Executor普通员工我今天要做什么，具体到每封邮件
HybridPrincipal + 有下属自己的待办 + 团队谁需要帮助
Leader / GM+3 层以上下属项目整体进度 + 团队健康度 + 决策事项

已实现：四问过滤在 todo-discover SKILL.md 中定义，角色适配在 USER.md 的 Role Type 中配置。

准备层：AI 做了什么（核心差异化）

这是 Concordia 和所有竞品的本质区别——不只是发现待办，而是提前帮你做了 80%。
信号类型AI 做什么用户做什么存储
需要回复的邮件/Teams草拟完整回复文本确认发送 / 改两句memory/todo-context/E{N}.md
需要审批/确认整理选项 + 利弊分析选一个同上
即将开始的会议生成会前简报（上次结论 + 今天要点 + 关联任务）带着去开会同上
需要评审的需求/文档分析可行性 + 给建议确认方向同上
需要交付的文档/方案起草框架/大纲补充细节同上
需要方向判断的决策列出选项（不替你决策）做决策同上

已实现：TODO-WRITER.md 定义了统一写入规范，每条待办自动创建 Context 文件并 spawn subagent 执行预处理。

能力边界（硬规则）：

  • ✅ AI 永远可以做：信息收集、草拟文本、数据分析、会议简报
  • ❌ AI 永远不能做：未经确认发送消息、涉及他人评价的决策、承诺时间线
  • ⚠️ 需要逐步授权：自动接受会议、自动回复固定模式邮件、自动更新 ADO

协作层：团队怎么一起工作

项目空间（对应产品的"工作台"Tab）：

一个项目空间 = 一个团队围绕一件事的所有信息：

模块内容来源
📋 概览项目健康度 + 进度条 + 阻塞 + 团队每人一行状态AI 从各信源综合
📄 知识AI 编译的结构化知识（来源溯源）Teams/邮件/文档 → AI 编译
📎 文件共享文档SharePoint/OneDrive 自动收集
💬 讨论摘要AI 从 Teams 自动整理的要点Teams 频道消息
✅ 团队任务任务 + 流转状态（Yana→Jian→Yana）ADO + AI 识别

项目空间从哪来：

  • AI 自动发现：分析 Teams 群 + 日历 + ADO，识别出"这是一个项目"
  • 手动创建：用户点"新建项目空间"，选择关联的 Teams 群/SharePoint/ADO

每个人看到什么：

  • 共享的：概览、知识、文件、讨论摘要 — 所有人一样
  • 个性化的：我的待办、和我相关的团队动态、需要我决策的事项

多碳多硅在这里的体现：

  • 每个成员的 AI 自动为项目空间贡献信息（🤖 AI 更新标注）
  • 项目 AI 发现跨人关联（"Yana 的阻塞会影响 Danxia 的 demo"）
  • AI 之间自动协调（Yana 的 AI 通知 Jian 的 AI 处理权限问题）

Team Skills 标准（参考 JiuwenClaw）：

把一次成功的项目协作封装为可复用的团队技能：

concordia-release/
├── SKILL.md          ← 发布团队：目标、成员、交付标准
├── roles/
│   ├── pm.md         ← PM：产品定义 + 排期 + 对外沟通
│   ├── tech.md       ← Tech：技术方案 + 实现 + 上线
│   ├── qa.md         ← QA：评测 + 标准化
│   └── marketing.md  ← Marketing：onboarding + demo
├── workflow.md       ← 每天 sync → 各自执行 → 阻塞上报 → 周五汇总
└── bind.md           ← 阻塞超 2 天自动升级给 GM

下次发布直接复用这个 Team Skill，不需要重新定义角色和流程。

学习层：越用越懂你

个人维度：

  • 用户接受了哪些草稿 → 学习写作风格
  • 用户修改了什么 → 学习偏好
  • 用户忽略了什么 → 降低优先级
  • 用户多次做同一类事 → 识别可自动化模式

团队维度：

  • 成功的协作流程 → 封装为 Team Skill
  • 反复出现的阻塞模式 → 自动预警

自主权等级（L1-L5）：

等级AI 行为升级条件
L1 辅助只提醒，不准备内容初始状态
L2 准备准备草稿，逐条确认使用 3 天
L3 建议执行准备+建议操作，批量确认连续 5 次未修改草稿
L4 代理执行常规事项自动处理，异常才汇报用户明确授权
L5 全自主完全代理日常工作企业级部署+管理员授权

升级是按任务类型的：回复 Shan 的周报可以 L4，回复 Dawei 的邮件只能 L2。

技术栈

运行时: OpenClaw Gateway (Node.js)
前端: 单文件 HTML (无框架，可升级为 React)
数据: TODO.md (文件) + memory/todo-context/ (Context 文件)
信号采集: Proactive Pipeline (YAML) + WorkIQ + Graph API
AI: Claude/GPT-4o (通过 Gateway LLM 路由)
部署: DevBox (当前) → Azure Static Web App (目标)
